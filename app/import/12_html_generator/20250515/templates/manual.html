lightbulb はじめに

この度は「Flask + Claude HTML Generator」をご利用いただきありがとうございます。このアプリケーションは、入力した指示（プロンプト）に基づいて、AI（Claude 3.7 Sonnet）が自動でHTMLコードを生成するツールです。

このマニュアルでは、基本的な使い方と、プログラムがどのように動いているのかを、専門的な知識がない方にも分かりやすく解説します。

touch_app 画面の見方と基本的な使い方

text_fields

1. システムプロンプト

画面左上にある大きなテキスト入力欄です。ここには、AIにHTMLを生成させる上での基本的な指示や前提条件を入力します。

例えば：

「あなたはプロのウェブデザイナーです。親切で丁寧な言葉遣いで回答してください。生成するHTMLは日本語で、Tailwind CSSのクラスを使用してください。」

初回アクセス時には、「HTML テキストを生成してください。HTML タグだけを返してください。」という汎用的な指示が入力されています。この内容は編集可能で、よく使う指示はここに固定しておくと便利です。

code システムプロンプト
HTML テキストを生成してください。HTML タグだけを返してください。
tips_and_updates ヒント: システムプロンプトはAIの役割や制約を設定するために使います。具体的なHTMLの内容はユーザープロンプトで指定しましょう。
chat

2. ユーザープロンプト

システムプロンプトの下にあるテキスト入力欄です。ここに、具体的にどのようなHTMLを作ってほしいかの指示を入力します。

例えば：

「会社の紹介ページを作ってください。会社名は『AIアシスト株式会社』で、事業内容は『AI導入コンサルティング』です。ヘッダー、主要コンテンツエリア、フッターの3部構成にしてください。」

細かい要望や具体的なデザイン指示をここに記述すると、より期待通りの結果が得られます。

person ユーザープロンプト
飲食店のランディングページを作成してください。店名は「和風カフェ 花柚」です。ヘッダー、メニュー紹介、店舗情報、フッターのセクションを含めてください。
tips_and_updates ヒント: 具体的で詳細なプロンプトほど、期待に近い結果が得られます。色やレイアウトの好みも指定してみましょう。
tune

3. 生成オプション

生成のカスタマイズオプションです。必要に応じて選択できます。

psychology

Thinking モード

チェックを入れると、AIがHTMLを生成する際の「思考プロセス」も一緒に表示されます。AIがどのように考えてそのHTMLを作ったのかを知りたい場合に便利です。ただし、生成に少し時間がかかることがあります。

search

Web検索 モード

チェックを入れると、AIがHTMLを生成する際にインターネット上の情報を検索し、それを参考にすることができます。最新情報や特定の固有名詞に関する情報を含めたい場合に役立ちます。こちらも生成に時間がかかることがあります。

settings 生成オプション

psychology Thinking モード

Claudeの思考プロセスを表示

search Web検索 モード

インターネットから最新情報を取得

lightbulb 開発中のプロジェクトではThinkingモードが非常に便利です。AIの思考過程を見ることで、より良いプロンプトの書き方を学べます。
play_arrow

4. HTMLを生成 ボタン

システムプロンプトとユーザープロンプトを入力し、必要に応じてオプションを選択したら、このボタンをクリックします。

AIがHTMLの生成を開始します。生成処理が始まると、ボタンの表示が「生成中...」に変わり、進行状況がわかるようになっています。

info

生成には数秒～数十秒かかる場合があります。特にThinkingモードやWeb検索モードを使用している場合は、より時間がかかることがあります。

autorenew 生成に時間がかかる場合は、シンプルなプロンプトから始めて、徐々に詳細を追加していくとよいでしょう。
wysiwyg

5. エディタエリア

HTMLが生成されると、画面右側のエリアに表示されます。このエリアには複数のタブがあり、用途に応じて表示を切り替えられます。

edit

WYSIWYGタブ

(What You See Is What You Get の略) 生成されたHTMLを見たまま編集できるモードです。ツールバーのボタン（H1, 太字, 画像挿入など）を使って、直感的に内容を修正できます。

code

コードタブ

生成されたHTMLのソースコードを直接編集できます。HTMLの知識がある方向けです。ここでの変更も他のタブに反映されます。

visibility

プレビュータブ

生成されたHTMLが実際にブラウザでどのように表示されるかを確認できます。編集はできません。

psychology

思考プロセスタブ

「Thinking モード」を有効にして生成した場合にのみ表示されます。AIがどのようにHTMLを考えて作成したのか、その思考過程を見ることができます。

fullscreen

エディタエリアの右上には「全画面表示」ボタンがあり、クリックするとエディタエリアが画面全体に広がります。もう一度クリックすると元に戻ります。

Claude で生成された HTML がここに表示されます...

sync エディタで行った変更は、他のすべてのタブにリアルタイムで反映されます。
download

6. ダウンロード ボタン

エディタエリアの下部（全画面表示時は右下）にある緑色のボタンです。クリックすると、現在エディタに表示されているHTMLコードを、「generated.html」というファイル名でダウンロードできます。

lightbulb

ダウンロードしたHTMLファイルは、そのままWebサイトとして公開したり、他のプロジェクトに組み込んだりすることができます。

ダウンロードされるファイルには、生成されたHTMLの他に、基本的なHTML構造（DOCTYPE宣言、head要素、必要なメタタグなど）も含まれています。

warning 利用上の注意

lightbulb

プロンプト作成のコツ

  • check_circle 具体的な指示を心がける（「会社紹介ページ」より「IT企業のランディングページで、サービス紹介・チーム紹介・お問い合わせフォームを含む」の方が良い結果に）
  • check_circle デザインの方向性を指定する（「モダンで明るい雰囲気」「シンプルで洗練されたデザイン」など）
  • check_circle 色やフォントの好みがあれば伝える（「青と白を基調としたデザイン」など）
gpp_maybe

注意事項

  • error 生成されたHTMLは必ずしも完璧ではありません。必要に応じて手動で修正してください。
  • error 複雑すぎる指示や曖昧な指示では、期待通りの結果が得られないことがあります。
  • error APIキー（ANTHROPIC_API_KEY）は秘密情報です。第三者に漏れないよう管理してください。
paid

APIコスト

Claude API の利用には、別途Anthropic社との契約と利用料金が発生する場合があります。利用規約をよくご確認ください。このアプリケーション自体の利用は無料ですが、AIの利用料金はユーザー様の負担となります。

security

セキュリティ

このアプリケーションは開発用サーバーで動作しています。本格的な公開環境で利用する場合は、Gunicornなどの本番用WSGIサーバーに切り替え、適切なセキュリティ対策を施すことを推奨します。

データフローの仕組み

1. ユーザーがプロンプトを入力して「HTMLを生成」ボタンをクリック
2. ブラウザがFlaskサーバーにリクエストを送信
3. FlaskサーバーがClaude APIにプロンプトを送信
4. Claude AIがHTMLを生成して返信
5. FlaskサーバーがHTMLをブラウザに送信
6. ブラウザがHTMLをエディタに表示

├── Dockerfile              # アプリを動かすための設計図 (Docker用)

└── templates/              # HTMLの「型紙」置き場
    ├── layout.html       # 全ページ共通の骨組み
    ├── index.html        # メインページの型紙
    ├── wysiwyg.html      # WYSIWYGエディタページの型紙
